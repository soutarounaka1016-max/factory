# DECISIONS.md

## D-001 読み取り専用
第1版は確認専用とする。誤操作と権限漏えいを避けるため、GitHubへの変更操作は実装しない。

## D-002 匿名GitHub API
公開リポジトリのREST APIをブラウザから取得する。秘密情報は不要で、取得不能項目は「未確認」とする。

## D-003 部分失敗を許容
各エンドポイントの失敗を個別記録し、1件のAPI失敗で画面全体を停止させない。

## D-004 前回データ
全APIが失敗した場合だけlocalStorageの前回取得結果を表示し、「前回取得データ」と明記する。秘密情報は保存しない。

## D-005 設定駆動
アプリ追加は `src/apps.js` の配列へ1件追加する方式とし、画面の再実装を不要にする。

## D-006 公開サブパス
既存の空リポジトリ `codex` を利用するため、公開サブパスを `/codex/` とする。

## D-007 古い公開の防止
Pagesワークフローは対象SHAが最新mainと一致する場合だけ公開し、concurrencyで古い実行をキャンセルする。

## D-008 GitHub Actions情報から公開状態を判定
匿名ブラウザから権限が必要になりやすいPages管理APIへ依存せず、公開リポジトリで取得できるActionsのCI・Pagesワークフローと各ジョブを分けて取得する。

## D-009 ローカルブラウザ確認の代替起動
実行環境の管理ポリシーがlocalhost表示を遮断するため、ローカル確認時だけビルド済みHTML・CSS・JavaScriptをPlaywrightへ直接読み込ませる。本番CIではViteのプレビューサーバーを実際に起動して確認する。

## D-010 CIと公開結果をコミットステータスへ記録
mainのCI結果を `ai-factory/ci`、Pages公開と公開URL確認を `ai-factory/pages` として記録する。これにより、公開APIから実行中・成功・失敗を判定できる。

## D-011 Pages初回有効化は管理設定として分離
GitHub Pagesの発行元を `GitHub Actions` にする初回設定は、リポジトリ管理権限を必要とする。秘密トークンを追加して自動化せず、GitHubの設定画面で一度だけ実施する。設定後のビルド、公開、公開URL確認は既存ワークフローへ任せる。
